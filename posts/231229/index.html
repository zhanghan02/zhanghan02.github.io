AI摘要：这是一个使用 Python 和第三方库 dashscope、frontmatter、pangu 等实现的 AI 摘要生成器。这个生成器主要的功能是从 Markdown 格式的文章中提取正文，然后使用阿里云的灵积模型服务进行摘要生成，最后将生成的摘要添加回原来的文章中。

本文AI摘要生成基于阿里云灵积模型服务模型使用

通义千问 qwen-turbo

服务简介

阿里云的这款服务名为灵积（DashScope），标榜为“模型服务”。简单来说，就是通过 Python / Java 的 SDK，或者是 HTTP 请求，提供在线的大模型 AI 服务。除了阿里自己的通义千问外，它还支持 LLaMa2、百川、Stable Diffusion、ChatGLM 等多种其他模型。

获取服务：

  1. 开通灵积服务
  2. 生成一个 key
  3. 通过 SDK 或 HTTP 请求，调用可选择的大模型

处理结果

总体来看生成摘要的成功率还是比较高的，其中较短的文章的摘要结果表现比较好，但是对于像《解析Transformer模型》这篇4.7k字数的博客来说，就出现了摘要之后输出大量重复文本的情况：

在实际添加的摘要中进行了手动修改，我对这种情况具体的出现原因比较好奇，因为在之前使用其他大模型进行比较长的上下文的对话过程中，也出现过类似情况，所以我去网上找了找原因，但是没找到，我又去问了问出现这种情况的模型，看看他们自己知道不知道，结果都是回答对不起，所以暂且搁置，待我去研究研究，问问大佬看看。

最终总计对14篇发布在我的个人博客上的文章进行生成摘要，有12篇成功生成，2篇失败的文章返回的错误码都是：Status code:400

文档对此的解释为：

数据检查错误，输入或者输出包含疑似敏感内容被绿网拦截

而这两篇一篇是论文笔记，一篇是联邦学习方向的综述，都是正常博客（—_—！）。

另外根据平台模型调用计费规则来看：模型API调用时按照输入、输出token 总数进行计费：对于中文文本来说，1个token通常对应一个汉字。对于英文文本来说，1个token通常对应3至4个字母。

我的博客总字数在43.3k，消耗的token是1998086/2000000，有点奇怪，仔细一看，是上午测试的时候用的，还没刷新：
